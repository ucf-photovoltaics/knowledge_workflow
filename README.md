
## Project Overview

A Python NLP pipeline for scientific literature analysis. It connects to a Zotero group library, extracts PDF text, runs NLP concept extraction over paper abstracts, and outputs structured CSV files for knowledge management.

The project targets a solar cell materials research domain (perovskites, encapsulants, etc.), though the pipeline is collection-agnostic.

## Environment Setup

This project uses `uv` for dependency management and requires Python 3.13.

```bash
# Install dependencies
uv sync

# Activate the virtual environment (Windows)
.venv\Scripts\activate

# Install the spaCy language model (required for V1 pipeline)
uv run python -m spacy download en_core_web_sm
```

## Running the Pipelines

The two main workflow scripts are run directly as Python scripts. Change `collection_name` inside the script before running.

```bash
# V1: spaCy + TF-IDF pipeline (outputs ontology matrix + concept frequencies)
uv run python knowledge_workflow_V1.py

# V2: KeyBERT pipeline (outputs concepts, rankings, and ontology matrix)
uv run python knowledge_workflow_V2.py

# Older bulk read script (also spaCy + TF-IDF, same structure as V1)
uv run python zotero_bulk_read.py
```

## Architecture

### Two Pipeline Variants

**V1 / `zotero_bulk_read.py`** — uses spaCy noun chunking + TF-IDF to surface concepts:
1. `get_collection_with_text()` — fetches paper metadata + PDF text from Zotero API
2. `extract_noun_phrases()` — spaCy noun chunks filtered by length
3. `build_ontology_matrix()` — TF-IDF selects top-N concepts; `match_sentences()` maps each concept back to source sentences in each abstract
4. Outputs: `ontology_<name>.csv` (paper × concept matrix), `frequencies_<name>.csv`

**V2 (`knowledge_workflow_V2.py`)** — uses KeyBERT (transformer-based) for concept extraction:
1. Same Zotero fetch layer
2. `extract_key_concepts()` — KeyBERT with MMR diversity, keyphrase ngrams 1–3
3. `build_concept_table()` → `build_concept_rankings()` — flat table then aggregation
4. `build_ontology_matrix()` — takes an explicit concept list, maps to source sentences
5. Step 9 optionally loads concepts from a prior `rankings_*.csv` to build a custom ontology
6. Outputs: `concepts_<name>.csv`, `rankings_<name>.csv`, `ontology_<name>.csv`, optionally `ontology_custom_<name>.csv`

### Custom Concept Ontology (CSV-driven)

Both versions support `build_ontology_from_csv(collection_dict, csv_path, column='concept')`, which uses a user-supplied CSV of domain terms instead of auto-extracted concepts. The `schemas/` directory holds reference CSVs for this purpose.
